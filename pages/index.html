1.0.0  eCQM Methodology with CQL and FHIRro Implementation Guide

 

1.1.0  Summary

The eCQM Methodology Implementation Guide defines a methodology for developing and evaluating Electronic Clinical Quality Measures (eCQM). In order to enforce the methodology this implementation guide defines a set of FHIR profiles, which are necessary to evaluate eCQMs. The profiles in this implementation guide provide a common foundation for building, sharing, and evaluating eCQMs across quality improvement efforts.

 

1.2.0  Scope

The scope of this guide is to provide guidance on the steps and processes involved in fostering a clinical quality measure specification into an electronic, executable representation. The development of clinical quality measure specifications are not included in the scope of this guide. For more information regarding the development of clinical quality measure specifications, visit the Electronic Clinical Quality Improvement (eCQI) Resource Center.

 

1.3.0  Lifecycle

The eCQM methodology defined within this guide adheres to the following development lifecycle:

 

1.3.1  Authoring

The initial phase of the eCQM development lifecycle is the authoring phase. The scope of this phase includes guidance on how to represent the logic outlined in the measure specification as executable expression logic. For the purposes of this guide, the expression logic will be represented in Clinical Quality Language (CQL).

For more information on this phase, visit the Authoring page in this guide.

 

1.3.2  Packaging

The second phase of the eCQM development lifecycle is the packaging phase. The scope of this phase includes developing the necessary Fast Healthcare Interoperability Resources (FHIR) needed to facilitate the FHIR $evaluate-measure operation.

For more information on this phase, visit the Packaging page in this guide.

 

1.3.3  Implementation

The third phase of the eCQM development lifecycle is the implementation phase. The scope of this phase includes the implementation requirements that must be satisfied in order to execute the FHIR $evaluate-measure operation. These requirements include access to a FHIR Knowledge Repository and a FHIR Measure Processing service.

For more information on this phase, visit the Implementation page in this guide.

 

1.3.4  Testing

The fourth phase of the eCQM development lifecycle is the testing phase. The scope of this phase includes guidance identifying test cases, methods for generating test data, and validating results. The testing phase is optional, but highly recommended.

For more information on this phase, visit the Testing page in this guide.

 

1.3.5  Publishing

The fifth phase of the eCQM development lifecycle is the publishing phase. The scope of this phase includes guidance on methods for documenting the requirements and assumptions made during the previous phases. For the purposes of this guide, publishing will be discussed in two forms: FHIR Implementation Guide and Health Quality Measure Format (HQMF).

For more information on this phase, visit the Publishing page in this guide.

 

1.3.6  Maintenance

The sixth phase of the eCQM development lifecycle is the maintenance phase. The scope of this phase includes updating the content generated in the initial phases based on feedback, changes to the measure specification, or some other trigger that initiates changes to the measure content.

For more information on this phase, visit the Maintenance page in this guide.
